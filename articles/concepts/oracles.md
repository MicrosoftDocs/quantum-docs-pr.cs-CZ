# <a name="quantum-oracles"></a><span data-ttu-id="6e2d0-182">Nejenom Oracle</span><span class="sxs-lookup"><span data-stu-id="6e2d0-182">Quantum Oracles</span></span>

<span data-ttu-id="6e2d0-183">Oracle $ O $ je operace "černého pole", která se používá jako vstup pro jiný algoritmus.</span><span class="sxs-lookup"><span data-stu-id="6e2d0-183">An oracle $O$ is a "black box" operation that is used as input to another algorithm.</span></span>
<span data-ttu-id="6e2d0-184">Tyto operace se často definují pomocí klasické funkce $ f: \\ { 0, 1 \\ } ^ n \to \\ { 0, 1 \\ } ^ m, $ která přebírá $ n $ -bit binárního vstupu a vytváří $ $ binární výstup m-bit.</span><span class="sxs-lookup"><span data-stu-id="6e2d0-184">Often, such operations are defined using a classical function $f : \\{0, 1\\}^n \to \\{0, 1\\}^m$ which takes an $n$-bit binary input and produces an $m$-bit binary output.</span></span>
<span data-ttu-id="6e2d0-185">Provedete to tak, že zaberete konkrétní binární vstup $ x = (X_ { 0 } , x_ { 1 } , \dots X_ { n-1 } ) $ .</span><span class="sxs-lookup"><span data-stu-id="6e2d0-185">To do so, consider a particular binary input $x = (x_{0}, x_{1}, \dots, x_{n-1})$.</span></span>
<span data-ttu-id="6e2d0-186">Qubit je možné označit jako $ \ket { \vec { x } } = \ket { X_ { 0 } } \otimes \ket { X_ { 1 } } \otimes \cdots \otimes \ket { X_ { n-1 } } $ .</span><span class="sxs-lookup"><span data-stu-id="6e2d0-186">We can label qubit states as $\ket{\vec{x}} = \ket{x_{0}} \otimes \ket{x_{1}} \otimes \cdots \otimes \ket{x_{n-1}}$.</span></span>

<span data-ttu-id="6e2d0-187">Můžeme se nejdřív pokusit o definování $ o $ , aby bylo $ o \ket { × } = \ket { f (x) } $ , ale to má několik problémů.</span><span class="sxs-lookup"><span data-stu-id="6e2d0-187">We may first attempt to define $O$ so that $O\ket{x} = \ket{f(x)}$, but this has a couple problems.</span></span>
<span data-ttu-id="6e2d0-188">V prvním $ případě $ může mít jazyk f jinou velikost vstupu a výstupu ( $ n \ne m $ ), takže při použití $ $ hodnoty O by došlo ke změně počtu qubits v registru.</span><span class="sxs-lookup"><span data-stu-id="6e2d0-188">First, $f$ may have a different size of input and output ($n \ne m$), such that applying $O$ would change the number of qubits in the register.</span></span>
<span data-ttu-id="6e2d0-189">Druhý, i když $ n = m $ , funkce nemusí být inverzi: Pokud $ f (x) = f (y) $ pro určitý $ x \ne y $ , pak $ o \ket { x } = o y, \ket { } $ ale $ o ^ \dagger o \ket { x } \ne o ^ \dagger o \ket { y } $ .</span><span class="sxs-lookup"><span data-stu-id="6e2d0-189">Second, even if $n = m$, the function may not be invertible: if $f(x) = f(y)$ for some $x \ne y$, then $O\ket{x} = O\ket{y}$ but $O^\dagger O\ket{x} \ne O^\dagger O\ket{y}$.</span></span>
<span data-ttu-id="6e2d0-190">To znamená, že nebudeme moct vytvořit sousedící operaci $ O ^ a \dagger $ Oracle musí mít pro ně definované sousedící.</span><span class="sxs-lookup"><span data-stu-id="6e2d0-190">This means we won't be able to construct the adjoint operation $O^\dagger$, and oracles have to have an adjoint defined for them.</span></span>

## <a name="defining-an-oracle-by-its-effect-on-computational-basis-states"></a><span data-ttu-id="6e2d0-191">Definování systému Oracle jeho účinkem na výpočetní bázi – stavy</span><span class="sxs-lookup"><span data-stu-id="6e2d0-191">Defining an oracle by its effect on computational basis states</span></span>
<span data-ttu-id="6e2d0-192">S oběma uvedenými problémy se můžeme vypořádat tím, že zavedete druhý registr $ m $ qubits a podržíte naši odpověď.</span><span class="sxs-lookup"><span data-stu-id="6e2d0-192">We can deal with both of these problems by introducing a second register of $m$ qubits to hold our answer.</span></span>
<span data-ttu-id="6e2d0-193">Pak definujeme účinek Oracle ve všech stavech výpočetního základu: u všech $ x \in \\ { 0, 1 \\ } ^ n $ a $ y \in \\ { 0, 1 \\ } ^ m $ ,</span><span class="sxs-lookup"><span data-stu-id="6e2d0-193">Then we will define the effect of the oracle on all computational basis states: for all $x \in \\{0, 1\\}^n$ and $y \in \\{0, 1\\}^m$,</span></span>

$$
\begin{align}
    <span data-ttu-id="6e2d0-194">O ( \ket { x } \otimes \ket { y } ) = \ket { x } \otimes \ket { y \oplus f (x) } .</span><span class="sxs-lookup"><span data-stu-id="6e2d0-194">O(\ket{x} \otimes \ket{y}) = \ket{x} \otimes \ket{y \oplus f(x)}.</span></span>
\end{align}
$$

<span data-ttu-id="6e2d0-195">Nyní $ o o = ^ \dagger $ podle konstrukce, proto jsme vyřešili obě předchozí problémy.</span><span class="sxs-lookup"><span data-stu-id="6e2d0-195">Now $O = O^\dagger$ by construction, thus we have resolved both of the earlier problems.</span></span>

> [!TIP]
<span data-ttu-id="6e2d0-196">>Chcete-li vidět, že o ^ $ = { \dagger } $ $ 2, = \boldone $ od $ \oplus b \oplus b = a $ pro všechny $ a, b \in \: :: No-Loc ({):: 0, 1 \: :: No-Loc (})::: $ .</span><span class="sxs-lookup"><span data-stu-id="6e2d0-196">> To see that $O = O^{\dagger}$, note that $O^2 = \boldone$ since $a \oplus b \oplus b = a$ for all $a, b \in \{0, 1\}$.</span></span>
<span data-ttu-id="6e2d0-197">>V důsledku toho se $ O \ket { x } \ket { a \oplus f (x) } = \ket { x } \ket { y \oplus f (x) \oplus f (x) } = \ket { x } \ket { y } $ .</span><span class="sxs-lookup"><span data-stu-id="6e2d0-197">> As a result, $O \ket{x} \ket{y \oplus f(x)} = \ket{x} \ket{y \oplus f(x) \oplus f(x)} = \ket{x} \ket{y}$.</span></span>

<span data-ttu-id="6e2d0-198">V důležitém případě je třeba definovat Oracle tímto způsobem pro každý výpočetní stav x y, a to $ \ket { } \ket { } $ i $ v případě $ , že funguje pro jakýkoliv jiný stav.</span><span class="sxs-lookup"><span data-stu-id="6e2d0-198">Importantly, defining an oracle this way for each computational basis state $\ket{x}\ket{y}$ also defines how $O$ acts for any other state.</span></span>
<span data-ttu-id="6e2d0-199">Z toho vyplývá bezprostředně z faktu, že $ o $ , stejně jako všechny operace v provozu, je lineární ve stavu, ve kterém funguje.</span><span class="sxs-lookup"><span data-stu-id="6e2d0-199">This follows immediately from the fact that $O$, like all quantum operations, is linear in the state that it acts on.</span></span>
<span data-ttu-id="6e2d0-200">Zvažte operaci Hadamard, například, která je definována v $ h \ket { 0 } = \ket { + } $ a $ h \ket { 1 } = \ket { - } $ .</span><span class="sxs-lookup"><span data-stu-id="6e2d0-200">Consider the Hadamard operation, for instance, which is defined by $H \ket{0} = \ket{+}$ and $H \ket{1} = \ket{-}$.</span></span>
<span data-ttu-id="6e2d0-201">Pokud chceme zjistit $ , jak h $ působí $ \ket { + } $ , můžeme použít tento $ h $ lineární,</span><span class="sxs-lookup"><span data-stu-id="6e2d0-201">If we wish to know how $H$ acts on $\ket{+}$, we can use that $H$ is linear,</span></span>

$$
\begin{align}
<span data-ttu-id="6e2d0-202">H \ket { + } & = \frac { 1 } { \sqrt { 2 } } h ( \ket { 0 }  +  \ket { 1 } ) = \frac { 1 } { \sqrt { 2 } } (h \ket { 0 } + h \ket { 1 } )\\\\</span><span class="sxs-lookup"><span data-stu-id="6e2d0-202">H\ket{+} & = \frac{1}{\sqrt{2}} H(\ket{0} + \ket{1}) = \frac{1}{\sqrt{2}} (H\ket{0} + H\ket{1}) \\\\</span></span>
           <span data-ttu-id="6e2d0-203">&= \frac{ 1 } { \sqrt { 2 } } ( \ket { + }  +  \ket { - } ) = \frac 12 ( \ket { 0 }  +  \ket { 1 }  +  \ket { 0 }  -  \ket { 1 } ) = \ket { 0 } .</span><span class="sxs-lookup"><span data-stu-id="6e2d0-203">& = \frac{1}{\sqrt{2}} (\ket{+} + \ket{-}) = \frac12 (\ket{0} + \ket{1} + \ket{0} - \ket{1}) = \ket{0}.</span></span>
\end{align}
$$

<span data-ttu-id="6e2d0-204">V případě definování našeho Oracle $ o $ můžeme obdobně použít jakýkoliv stav $ \ket { \psi } $ na $ n + m $ qubits, jak je možné zapsat</span><span class="sxs-lookup"><span data-stu-id="6e2d0-204">In the case of defining our oracle $O$, we can similarly use that any state $\ket{\psi}$ on $n + m$ qubits can be written as</span></span>
